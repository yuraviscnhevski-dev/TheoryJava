## 1. Spring Security: что такое авторизация и аутентификация?

Это два фундаментальных понятия безопасности:

| Термин | Определение | Пример |
|--------|-------------|--------|
| **Аутентификация** | Процесс проверки личности пользователя | Ввод логина и пароля |
| **Авторизация** | Процесс проверки прав доступа | Доступ к `/admin` только для ADMIN |

**Простыми словами:**
- Аутентификация: **"Кто ты?"**
- Авторизация: **"Что тебе можно?"**

## 2. Объекты Principal, Authorities, Authentication

Ключевые интерфейсы Spring Security:

| Интерфейс | Назначение |
|-----------|------------|
| **Principal** | Базовое представление пользователя |
| **Authentication** | Объект аутентификации |
| **GrantedAuthority** | Разрешения/роли пользователя |

**Authentication** содержит:
- `getPrincipal()` — идентификатор пользователя
- `getCredentials()` — учетные данные (пароль)
- `getAuthorities()` — коллекция разрешений
- `isAuthenticated()` — статус аутентификации

## 3. Чем отличается InMemoryAuthentication от BasicAuthentication?

Это разные вещи:

| Критерий | InMemoryAuthentication | BasicAuthentication |
|----------|------------------------|---------------------|
| **Что это** | Способ **хранения** пользователей | Способ **передачи** учетных данных |
| **Суть** | Хранение учетных записей в памяти приложения | HTTP-аутентификация через заголовок |
| **Где хранятся данные** | В памяти (оперативная память) | Не хранятся, передаются с каждым запросом |

**InMemoryAuthentication — пример:**
```java
@Bean
public UserDetailsService users() {
    UserDetails user = User.builder()
        .username("user")
        .password("{noop}password")
        .roles("USER")
        .build();
    return new InMemoryUserDetailsManager(user);
}
BasicAuthentication — пример:

java
http.httpBasic(); // Включаем Basic-аутентификацию

4. Как мы можем добавить секьюрность к контроллеру? (2 способа)
Способ 1: Глобальная конфигурация
java
http.authorizeRequests()
    .antMatchers("/public/**").permitAll()
    .antMatchers("/admin/**").hasRole("ADMIN")
    .anyRequest().authenticated()
Способ 2: Аннотации на методах
java
@PreAuthorize("hasRole('ADMIN')")
@GetMapping("/users")
public List<User> getUsers() { ... }

6. Как работают каскады для таблиц и какие они бывают?
Типы каскадов в JPA:

Тип	Описание
ALL	Все операции каскадируются
PERSIST	При сохранении родителя сохраняются дети
MERGE	При обновлении родителя обновляются дети
REMOVE	При удалении родителя удаляются дети
Пример:

java
@OneToMany(mappedBy = "user", cascade = CascadeType.ALL)
private List<Order> orders;

10. Что такое responseBody, requestBody, ResponseEntity?
@RequestBody
Получение JSON из запроса:

java
@PostMapping("/users")
public User create(@RequestBody User user)
@ResponseBody
Отправка JSON в ответе:

java
@GetMapping("/user")
@ResponseBody
public User getUser()
ResponseEntity
Полный контроль над ответом:

java
return ResponseEntity.ok(user);
return ResponseEntity.notFound().build();
return ResponseEntity.status(201).body(user);

12. Чем аннотация RestController отличается от Controller?
@Controller	@RestController
Возвращает имя представления (view)	Возвращает объект → JSON
Для веб-страниц (HTML, JSP)	Для REST API
Нужна @ResponseBody на методах	Не нужна, автоматически
Пример:

java
@Controller  // для HTML страниц
public class WebController { }

@RestController  // для JSON API
public class ApiController { }
