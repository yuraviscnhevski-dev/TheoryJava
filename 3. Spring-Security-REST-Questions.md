## 1. Spring Security: что такое авторизация и аутентификация?

Это два фундаментальных понятия безопасности, которые часто путают:

| Термин | Определение | Пример |
|--------|-------------|--------|
| **Аутентификация** | Процесс проверки личности пользователя (кто ты?) | Ввод логина и пароля, проверка по отпечатку пальца |
| **Авторизация** | Процесс проверки прав доступа (что тебе можно?) | Доступ к странице `/admin` только для роли ADMIN |

**Простыми словами:**
- Аутентификация отвечает на вопрос **"Кто ты?"**
- Авторизация отвечает на вопрос **"Что тебе можно?"**

В Spring Security эти процессы разделены и настраиваются отдельно. Аутентификация происходит через `AuthenticationManager`, а авторизация — через конфигурацию `HttpSecurity`.

## 2. Объекты Principal, Authorities, Authentication

Это ключевые интерфейсы Spring Security, которые нужно понимать:

| Интерфейс | Назначение | Ключевые методы |
|-----------|------------|-----------------|
| **Principal** | Базовое представление пользователя (из Java SE) | `getName()` |
| **Authentication** | Основной объект аутентификации в Spring | `getPrincipal()`, `getCredentials()`, `getAuthorities()`, `isAuthenticated()` |
| **GrantedAuthority** | Представляет разрешение/роль пользователя | `getAuthority()` |

**Подробнее об Authentication:**

`Authentication` — это центральный интерфейс, который представляет:
- Запрос на аутентификацию (еще не проверенный пользователь)
- Аутентифицированного пользователя (уже проверенный)

Основные методы:
- `getPrincipal()` — возвращает идентификатор пользователя
- `getCredentials()` — возвращает учетные данные (пароль)
- `getAuthorities()` — возвращает коллекцию разрешений
- `isAuthenticated()` — показывает, прошел ли пользователь аутентификацию

После успешной аутентификации объект `Authentication` помещается в `SecurityContext`, который хранится в `SecurityContextHolder`.

## 3. Чем отличается InMemoryAuthentication от BasicAuthentication?

Это разные концепции, которые часто путают:

| Критерий | InMemoryAuthentication | BasicAuthentication |
|----------|------------------------|---------------------|
| **Что это** | Способ **хранения** пользователей | Способ **передачи** учетных данных |
| **Суть** | Хранение учетных записей в памяти приложения | HTTP-аутентификация через заголовок `Authorization` |
| **Где хранятся данные** | В памяти (оперативная память) | Не хранятся, передаются с каждым запросом |
| **Когда использовать** | Тестирование, прототипы | API, сервисы |

**InMemoryAuthentication — пример:**

```java
@Configuration
public class SecurityConfig {
    
    @Bean
    public UserDetailsService users() {
        UserDetails user = User.builder()
            .username("user")
            .password("{noop}password")
            .roles("USER")
            .build();
        return new InMemoryUserDetailsManager(user);
    }
}
BasicAuthentication — пример:

java
@Configuration
public class SecurityConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .authorizeRequests()
                .anyRequest().authenticated()
                .and()
            .httpBasic();
        return http.build();
    }
}
4. Как мы можем добавить секьюрность к контроллеру? (минимум 2 способа)
Способ 1: Глобальная конфигурация через HttpSecurity

java
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .authorizeRequests()
                .antMatchers("/public/**").permitAll()
                .antMatchers("/admin/**").hasRole("ADMIN")
                .anyRequest().authenticated()
                .and()
            .httpBasic();
        
        return http.build();
    }
}
Способ 2: Аннотации на уровне методов

java
@RestController
@RequestMapping("/api/users")
public class UserController {
    
    @GetMapping("/profile")
    @PreAuthorize("isAuthenticated()")
    public User getProfile() {
        return userService.getCurrentUser();
    }
    
    @GetMapping
    @PreAuthorize("hasRole('ADMIN')")
    public List<User> getAllUsers() {
        return userService.findAll();
    }
}
5. Связи таблиц many-to-many, one-to-one
One-to-One (Один к одному)

java
@Entity
public class User {
    @Id
    @GeneratedValue
    private Long id;
    
    @OneToOne(mappedBy = "user")
    private Passport passport;
}

@Entity
public class Passport {
    @Id
    @GeneratedValue
    private Long id;
    
    @OneToOne
    @JoinColumn(name = "user_id")
    private User user;
}
Many-to-Many (Многие ко многим)

java
@Entity
public class Student {
    @Id
    @GeneratedValue
    private Long id;
    
    @ManyToMany
    @JoinTable(
        name = "student_course",
        joinColumns = @JoinColumn(name = "student_id"),
        inverseJoinColumns = @JoinColumn(name = "course_id")
    )
    private List<Course> courses;
}

@Entity
public class Course {
    @Id
    @GeneratedValue
    private Long id;
    
    @ManyToMany(mappedBy = "courses")
    private List<Student> students;
}
6. Как работают каскады для таблиц и какие они бывают?
Типы каскадов:

Тип	Описание
ALL	Все операции каскадируются
PERSIST	При сохранении родителя сохраняются дети
MERGE	При обновлении родителя обновляются дети
REMOVE	При удалении родителя удаляются дети
REFRESH	При обновлении родителя обновляются дети
DETACH	При отсоединении родителя отсоединяются дети
Пример:

java
@OneToMany(mappedBy = "user", cascade = CascadeType.ALL)
private List<Order> orders;

8. REST-сервисы. Их преимущества и недостатки
REST (Representational State Transfer) — архитектурный стиль для создания веб-сервисов.

Преимущества:

Простота (стандартные HTTP-методы)

Масштабируемость (stateless)

Гибкость (JSON, XML)

Кэширование

Универсальность

Недостатки:

Избыточность данных

Сложность со связанными данными

Нет строгой спецификации

10. Что такое responseBody, requestBody, ResponseEntity?
@RequestBody — получает JSON из запроса и превращает в Java-объект

java
@PostMapping("/users")
public User create(@RequestBody User user)
@ResponseBody — отправляет Java-объект как JSON в ответе

java
@GetMapping("/user")
@ResponseBody
public User getUser()
ResponseEntity — позволяет управлять статусом, заголовками и телом ответа

java
return ResponseEntity.ok(user);
return ResponseEntity.notFound().build();
return ResponseEntity.status(201).body(user);
11. Что такое AJAX/fetch?
AJAX — технология для асинхронного обмена данными с сервером без перезагрузки страницы.

Fetch API — современный способ делать запросы:

javascript
fetch('/api/users')
    .then(response => response.json())
    .then(data => console.log(data))
